### 1.为什么422.645 格式化的时候为  $422.64

`format("$,.2f")(data)`

#### 问题原因

在 JavaScript 中，422.645 实际上并不是精确的 422.645：

```js
console.log(422.645);           // 422.645
console.log(422.645.toPrecision(17)); // "422.64499999999998"
```

由于二进制浮点数的表示限制，422.645 在内存中实际存储为 422.64499999999998...。

当 d3-format 对这个值进行四舍五入时：

- 第三位小数是 4（而不是 5）

- 所以结果是 $422.64 而不是 $422.65
